**Universal Capture. Transform. Send.**

One pipeline engine on Cloudflare's edge. Multiple surfaces for data capture. One YAML format for processing. Fast, durable, fire-and-forget. Evolving into a capability orchestrator where pipelines invoke AI services, render engines, and external tools through a unified cost-aware routing layer.

## What is Scramjet?

Scramjet is a pipeline platform that captures data from any source, transforms it through composable operators on Cloudflare's edge, and sends results anywhere. Every operation — from a simple RSS digest to a scheduled multi-step AI pipeline — runs as a YAML pipeline with durable execution, automatic retries, and plan-based resource limits.

Scramjet is also becoming a capability orchestrator. The `invoke` operator (RFC-SJ-001) lets pipelines call external capabilities — text-to-speech, video rendering, image generation — through API Mom, a capability registry that handles routing, cost tracking, and provider selection. This turns Scramjet from a data pipeline into a general-purpose workflow engine where each step can be an AI call, a render job, or any service with a cost.

### How It Works

```
Capture (any surface)
    |
Edge Engine (Cloudflare Workers)
    | YAML pipeline: fetch -> transform -> invoke -> output
    | Durable execution via Workflows
    | Capability invocation via API Mom
    | File buffering via Media Store
    |
Send (webhook, Slack, R2, API, email, Discord, Telegram, Notion, GitHub)
```

## Pipeline Examples

### Data pipeline

```yaml
name: tech-digest
schedule: "0 7 * * *"

fetch:
  - type: rss
    url: https://hnrss.org/frontpage
  - type: reddit
    subreddit: programming
    sort: top
    limit: 25

transform:
  - dedup: url
  - ai:
      model: haiku
      prompt: "Score relevance to serverless/edge, 1-10"
      output: relevance
  - filter: "relevance.score > 7"
  - sort: relevance.score desc
  - limit: 10

output:
  - json
  - slack:
      webhook: "{{ secrets.slack_webhook }}"
```

### Capability pipeline (invoke operator)

```yaml
name: render-video
schedule: manual

fetch:
  - type: api
    url: "{{ secrets.content_api }}/scripts/pending"

transform:
  - invoke:
      capability: text-to-speech
      input: { text: "{{ item.narration }}", voice: "christopher" }
      output: audio_key
      cache: true
  - invoke:
      capability: video-render
      input: { clips: "{{ item.clips }}", audio_key: "{{ item.audio_key }}" }
      output: video_key
      budget_cents: 50

output:
  - r2:
      key: "videos/{{ item.id }}.mp4"
```

## Capability Architecture

Scramjet is evolving from a pipeline engine into a three-layer capability orchestrator:

```
Layer 3: Scram-Jet (pipeline orchestration)
         YAML pipelines, scheduling, durable execution
              |
Layer 2: API Mom (capability registry + routing)
         Cost tracking, provider selection, budget enforcement
              |
Layer 1: Capability providers
         Local machines, rented GPUs, commercial APIs (OpenAI, ElevenLabs, Remotion)
```

The `invoke` operator (RFC-SJ-001) bridges Layer 3 and Layer 2. A pipeline step like `invoke: { capability: text-to-speech }` routes through API Mom's CapabilityRegistryDO, which resolves the best provider based on cost, latency, and availability. Results flow back as pipeline items. Media Store provides universal file buffering between steps.

See `hq/rfcs/RFC-SJ-001-invoke-capability-operator.md` for the full design.

## Architecture

### Stack

- **Runtime:** Cloudflare Workers + Durable Objects (SQLite) + Workflows + Queues + Browser Rendering + R2
- **HTTP Framework:** Hono
- **Validation:** Zod (pipeline schemas in `@scramjet/shared`)
- **Auth:** Better Auth (cross-subdomain `.scramjet.io` cookies) + API keys (SHA-256)
- **AI Agent Interface:** MCP server (`@modelcontextprotocol/sdk` + `agents`)
- **Dashboard:** Vite + React 19 + TanStack Router, served by Hono Worker
- **Language:** TypeScript only

### Cloudflare Bindings

| Binding | Type | Purpose |
|---------|------|---------|
| `PIPELINE_STORE` | Durable Object | Per-pipeline SQLite state (items, dedup, runs, batches) |
| `PIPE_WORKFLOW` | Workflow | Durable pipeline execution with automatic retries |
| `DB` | D1 | Central registry (accounts, pipelines, api_keys, secrets, audit log) |
| `ARCHIVE` | R2 | Temporary output buffer (TTL-based) |
| `AI` | Workers AI | AI transform operator |
| `BROWSER` | Browser Rendering | Puppeteer-based JS-rendered page scraping |
| `EVENTS_QUEUE` | Queue | Event-driven pipeline triggers |

### Execution Flow

```
AI Agent -> MCP Server -> Validate YAML -> Store in D1
                                           |
                        Cron (*/1 min) -> PipeWorkflow (CF Workflow)
                                           |
                        1. Resolve secrets (decrypt from D1)
                        2. Fetch (parallel, all sources)
                        3. Store items in PipelineStore DO (batch offloading)
                        4. Transform pipeline (sequential)
                        5. Output (parallel, all sinks)
                        6. Persist final items + complete run record
```
